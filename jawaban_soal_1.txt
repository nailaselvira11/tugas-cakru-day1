Apa itu merge conflict dan cara menanggulanginya?
Merge conflict adalah conflict yang terjadi saat beberapa developer bekerja pada repositori yang sama, developer 1 ingin mengubah file dalam repositori tersebut dengan menge-clone dan mengganti isi file tersebut, selanjutnya meng-commit hasil changesnya dan memasukkan ke branch nya, namun sesaat setelahnya developer 2 ingin mengganti isi file tersebut juga dengan mengubah di line yang sama juga lalu di commit hasil changes nya dan memasukkan ke branch miliknya. Nantinya git akan memberi tahu file mana yang sedang terjadi merge conflict. Ada 4 pilihan untuk resolve masalahnya, accept current change, accept incoming change, lalu accept both change, dan compare changes. Komunikasikan dengan developer itu dan pilih yang paling benar. Setelahnya git add., git commit -m merge resolved.
