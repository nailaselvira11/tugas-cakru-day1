Apa itu fork dan pull request github
Saat ingin berkontribusi atau mengubah kode di suatu projek besar dari public repository milik developer A (Upstream repository), developer B akan melakukan forking (Forked repository yang merupakan copy dari upstream repository). Forked repository ini terlepas dari upstream repository sehingga developer B bisa langsung cloning ke git, lalu membuat changes di projek tersebut. Setelah membuat changes, developer B akan push changesnya ke forked repository. Selanjutnya developer B akan melakukan pull request dari forked repository ke upstream repository milik developer A. lalu developer A bertugas untuk mereview pull request itu selanjutnya menyetujui dan merge jika perubahan yang dibuat oleh developer B sesuai.
